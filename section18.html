185. Introducing Object Literals

JS Objects. Our second data structure. UNlike arrays which is about ordering data ie a 1st, a 2nd, a 3rd. With objects our data is stored using key value pairs/properties. A property is just two pieces of information and then a key and then a value , and they form a pair . An object is just a bunch of these pairs.

  • Objects are collections of properties
  • Properties are a key-value pair
  • Rather than accessing data using an index, we use custom keys
So remember
PROPERTY = KEY + VALUE
We use the key to get the data back out. example of key-value pairs
KEY-VALUE pairs
username: - 'crazyCatLady'
upvotes - 7
text - 'great post'

186. Creating Object Literals

THe term object in JS is a bit of a loaded term. When we do typeof[] we get "object". From now on when we say Object literal we are refering to these key vlaue pair data stores, this data structure, that we create using curly braces. Here we are going to model a person. THink of the below creates an empty object. Note, objects are also reference types. Rememeber there are primitve types such as numbers strings booleans, undefined, null, or symbol. If it is anything else then it's a reference value (i.e. typeof object) empty object

const person = {}
const person = {firstName: 'Mick', lastName: 'Jagger'}
So above we now have two value pairs. Each pair is a key, a colon, and then a value, seperated by commas. Doing it on seperate lines for longer objects makes it easier to read.

187. Accessing Data Out Of Objects

Two ways of getting information out of an objects. The first we can use square brackets

person['firstName']
'Antony'

The second way is

person.lastName
'Gregory'

Both ways return a string. But there is an important distinction. To help us understand what is going on let's talk about what happens when we make an object. When we make a key: 'value', not only the value is returned as a string but the key is also converted to a string (except for symbols, which we will cover later). So if we create a some properties that rank certain years as good or bad i.e.

const yearRank = {1999: "good", 2020: "bad"};
