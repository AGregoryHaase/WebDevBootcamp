162. Equality: Triple Vs. Double Equals

aka. WEIRDNESS

The double equals (==)

  • Checks for equality of value, but not equality of type.
  • It coerces both values to the same type and then compares them.
  • This can lead to some unexpected results!
We then go into the console and type
1 == 1
and we get
true
as a response.

We then hit

1 == `1`
and we get the same response
true

Now obviously 1 the number and "1" the string are not the same thing.

The weirdness continues

Here are a few examples.

Here comes the weirdness!!!

  1. 1 != 2
    true
  2. 1 != `2`
    true
  3. 1 != `1`
    false
  4. 1 == `1`
    true
  5. 1 !== `1`
    true
  6. 1 === `1`
    false
Okay so there is some strange behavior going on here. First of all 1 != 2 is true which you would expect. As is 1 != "2".

Where it gets weird is

1 != `1`
false
and
1 == `1`
true
but then
1 == `1`
true
so you would assume that !== would be false, however
1 !== `1`
true
The key is in the names. !== is strict non equality whilst === is strict equality.

For now, !== and === are the two operators we should use. Colt actually says use triple equals always.

Logical NOT

The final logical operator we are looking at is the NOT operator. !expression returns true if expression is false.

  • !null returns true
  • ! (0 === 0 returns false
  • !3 (3 <= 4) returns false

So basically what happens is this operates on the expression. If something is true it turns it false and vice versa. It negates the value. If we put the ! at the front of an expression then it will return true if something is false.

We get an example of asking a user to enter a name.

const firstName = prompt("Enter first name");
THen we could write the following

if (firstName === ""){
firstName = prompt("try again")
}
however we can be more concise by using

if (!firstName){
firstName = prompt("try again")
}
Let's go to the JS file and try it out. COol that works. Next we have an example from the amusement park pricing

The below will only print out if someone is between 5 and 65


if (!(age >= 0 && age < 5 || age >= 65)) {
console.log("You're an adult")

So the ! turns finding someone who is free into finding the reverse, an adult who is paying.

173. The Switch Statement is...A Lot

const sday = 34;

console.log("wed")
