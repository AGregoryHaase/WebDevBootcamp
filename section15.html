153. Indices & Length

STRINGS ARE INDEXED

STRINGS are indexed, every letter in a string has a corresponding number. i.e. Chicken

  • C=0
  • H=1
  • I=2
  • C=3
  • K=4
  • E=5
  • N=6
We have these indeces. If we add [n] after the string variable and then put in a number we are then accessing the nth number in that string. So then if we have
let name = "Antony Haase"
and then we do
name[5]
then it should return the 5th letter, counting from A as zero. And, yep, I just tested that and it returned
'y'

If we request a value that doesn't exist. E.g. if we put in name[20] we would have "undefined" returned to us. Undefined is a special value, a primitive value and is JS's way of telling us that there is nothing there. WE cover more on this later.

We usually don't use these indices in this way. Typically we will check the first character of a string. Colt gives an example of a phone number. We can write logic to check if someone has used a perenthesis to start the number.

Length

every string has a special property of length. So if I go into the console and type
name.length
then it will return the length of the name
12
NOTE, this is not the same as the highest index as we are starting the count of the length from 1, not zero.

concatenation

If we write

"poop" + "a" + "scoop"
we get
poopascoop

We can create two variables like

let name = "Antony Haase";
let random = "wears big bras";
and then go
name + random
and we would get back
Antony Haasewears big bras

How do I put in a space? you have to do the space in

" "
so we go
name + " " + random
this way we get
Antony Haase wears big bras
Cool Beans.

Finally, if we go
1 + "hi"
then JS tries to coerce them into a common type and the result is a string.

156. String Template Literals - SUPER USEFUL

TEMPLATE LITERALS. First of all let's look at life without template literals. we would have

let product = "artichoke";
Then
let price = 2.25;
and then
let qty = 5;
and what we want to do is print a string that says something like - you bought x artichoke and the total is price times quantity. so for this we can concatonate
'you bought ' + qty + ' ' + product + '. Total is: ' + price*qty;
'you bought 5 artichoke. Total is: 11.25'
While this does work, it is a lot of work. Thankfully we now have template literals.

With template literals, a newer syntax, where we can embed an expression inside the string and that expression will be evaluated and turned into a string. The example given is

`I counted ${3 + 4} sheep`;
'I counted 7 sheep'
There's a couple of things we must know. We hve to use the backtick (`) character. We can actually jsut create a string using backticks. What is special about backticks is that when they are combined with ${}, whatever is inside the curly braces will be treated as an expression and evaluated.

I just did

`hey ho, what do you know? Do you know what 59995 divided by 823 is? Sure, it's ${59995 / 823}`;
"hey ho, what do you know? Do you know what 59995 divided by 823 is? Sure, it's 72.89793438639126"
Going back to what we made earlier, without string template literals. we would have
`you bought ${qty} ${product}. Total is: $${price * qty}`;
'you bought 5 artichoke. Total is: $11.25'
Just to note, we used a double $$ as the first one would escape our string due to it being part of the syntax.

We can also call a method inside of the curly brackets

`you bought ${qty} ${product.toUpperCase()}. Total is: $${price * qty}`;
'you bought 5 ARTICHOKE. Total is: $11.25'

157. Undefined & Null

Both are primative values and can be a bit confusing at first. It can seem as though they have negative connotations. It might seem like they refer to a problem. Both are single value. Both Null and undefined are values.

Undefined is JS's way of saying "I don't know what you want"

Null is intentionally set at that value.

  • Null
    • "Intentional absence of any value"
    • Must be assigned
  • Undefined
    • Variables that do not have an assigned value are undefined.
